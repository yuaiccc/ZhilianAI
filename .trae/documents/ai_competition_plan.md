# 智联招聘“全国AI创新大赛”参赛作品实施计划

## 1. 核心定位与目标
- **参赛赛道匹配**：以“用户（求职者）视角”切入，破解“规划无方向、简历无头绪、投递无回复”的求职困境。
- **产品核心形态**：**C端“多Agent会诊面试与辅导”平台**。
- **交互创新点**：**群聊会诊模式**。不仅有高压的模拟面试，还首创“评审过程透明化”——面试结束后，向求职者展示HR、业务专家、成长导师三方Agent的“内部讨论与争辩”，打破面试反馈黑盒。
- **技术实现方案**：前后端分离，后端接入真实的本地大模型 (Ollama + Qwen2.5) 实现动态、实时的对话与评估生成。

## 2. 关于《AI大赛脱敏数据.xlsx》的绝佳利用方案
表格中包含了极其丰富的真实岗位画像（如：职位名称、薪资、关键词、详细描述等）。
在本项目中，我们将它作为**“真题题库（JD Database）”**：
1. **岗位广场（JD直选）**：在C端首页，求职者无需自己编造岗位，而是直接从表格数据中挑选（或搜索）一个真实的JD进行“投递”。
2. **Agent的“最强上下文”**：将选中JD的`职位描述`、`职位关键词`、`薪资`等字段作为 Prompt 注入给大模型。
   - **HR Agent**：重点比对候选人简历与表格中的`学历要求`、`年限要求`，并考察求职动机。
   - **Biz (业务) Agent**：针对表格中的`职位关键词`和`职位描述`，生成硬核、精准的专业技能连命题。
   - **Growth (成长) Agent**：评估候选人潜力是否匹配该岗位的长期发展。

## 3. 实施步骤 (Implementation Steps)

### Step 1: 数据处理与本地化 (Data Preparation)
- **任务**：将 `AI大赛脱敏数据.xlsx` 解析并转换为前端/后端易于读取的 `jd_database.json`。
- **产出**：结构化的真实岗位库，支持前端随机抽取或列表展示。

### Step 2: 搭建 Node.js 中间层服务 (Backend Orchestration)
- **任务**：由于需要接入本地大模型 (Ollama) 并协调三个 Agent，纯前端直接请求会比较混乱且难以控制提示词。我们将使用 Express.js 创建一个轻量级服务端 (`server.js`)。
- **API设计**：
  1. `/api/chat`：接收候选人的简历、选择的JD和聊天上下文，调度指定的 Agent 进行回复。
  2. `/api/arbitration`：触发三个 Agent 的“群聊内部讨论”和生成“最终诊断报告”。

### Step 3: 开发 C端“群聊会诊”界面 (Frontend `candidate.html`)
- **UI设计**：复用现有的 `styles.css` 风格（深蓝科技风），构建类似微信/钉钉的群聊界面。
- **核心流程**：
  1. **入场**：用户选择 JD -> 输入个人简历亮点 -> 点击“开始面试”。
  2. **群面环节**：界面左侧是候选人发言，右侧是三个Agent头像交替提问。
  3. **黑盒开启（会诊环节）**：面试结束，界面进入“评审模式”，展示三个Agent之间的自动对话（例如HR说薪资太高，Biz说技术很强）。
  4. **通关攻略**：最终弹出一个类似 B 端面板的“诊断与提升报告”，给出具体的 `action_suggestions`。

### Step 4: 提示词工程与效果调优 (Prompt Engineering)
- **任务**：为 Ollama 编写精准的 System Prompts。
  - 赋予 HR Agent “严谨、看重稳定性和性价比” 的人设。
  - 赋予 Biz Agent “专业、挑剔技术细节” 的人设。
  - 赋予 Growth Agent “看重学习能力、鼓励式” 的人设。
  - 设计合议阶段的 Prompt，让它们能基于面试记录产生分歧和最终共识。

## 4. 商业闭环（可写进参赛PPT）
- **痛点解决**：为C端提供“带辅导的高质量模拟面试”，为B端（企业）提供“经AI多维验证的优质候选人”。
- **可落地性**：利用真实大厂JD库+开源大模型，成本极低，完全可以作为智联招聘App的一个VIP增值功能或学生专区引流工具。

## 5. 验收标准
- 用户能从提取的 Excel 数据中选择岗位。
- 用户能与本地 Ollama 模型驱动的三个 Agent 顺畅对话。
- 面试结束后能看到 Agent 的合议讨论过程，并获得结构化的改进建议。
